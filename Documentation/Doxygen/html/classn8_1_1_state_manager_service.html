Detailed Description

State_Manager handles the state stack and holds pointers to each registered game state.

The singleton pattern is used so there is a single state stack and access point to state information.

Stores a pointer to a State subclass in a map. If there are no states currently in the state stack, the state being added is pushed onto the stack. If a state with the specified identifier has previously been added, the new state is not added to the map.
